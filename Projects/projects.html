Automated SOC Defense System

Wazuh TheHive Cortex Shuffle

This project focuses on building a fully automated Security Operations Center (SOC) pipeline. By integrating Wazuh (SIEM) with TheHive (Case Management), I successfully reduced the Mean Time to Respond (MTTR) by 40% for common alert types.

1. The Problem

Manual triage of high-volume alerts leads to "alert fatigue." Analysts were spending too much time copying data between tools rather than investigating the actual threats.

2. The Solution Architecture

I designed a middleware solution using Shuffle to act as the glue between the detection engine and the response platform.

  • Wazuh: Detects threats on endpoints.
  • Shuffle: Receives the alert JSON, enriches it with VirusTotal data.
  • TheHive: Creates a case automatically with all evidence attached.

3. Implementation Details

Below is the automation logic used to parse the incoming webhook from Wazuh. 

# Example: Shuffle Workflow Logic
if alert.level >= 10:
    case_id = thehive.create_case(
        title=f"High Severity: {alert.rule.description}",
        description=alert.full_log,
        severity=3
    )
    cortex.analyze(case_id, alert.source_ip)

4. Key Results

The system now handles 90% of Level 1 triage without human intervention.
